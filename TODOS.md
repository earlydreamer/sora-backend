## Critical (데이터 손실 / 상태 손상)

- [x] **[C1] window name double-prefix** — `spawn-worker`가 `worker-${worker_id}-...` 생성 시
  `worker_id`가 이미 `worker-001`이라 `worker-worker-001-...` 생성. `dashboard`, `recover-session`의
  slug 파싱이 첫 워커부터 깨짐. (`spawn-worker:126`)

- [x] **[C2] next_worker_id() 경쟁 조건** — `find|wc -l` 방식으로 ID 계산. 동시 spawn 시 동일 ID 선택.
  파일 삭제 후 ID 재사용. max suffix 방식으로 변경 필요. (`lib.sh:64`)

- [x] **[C3] recover-session retry 좀비** — `--auto-fix` 시 old failed JSON의 `retry_count`만 증가 후
  새 worker를 spawn. old record는 `failed`로 방치 → 반복 시 worker 무한 증식.
  기존 worker JSON을 `retrying` 상태로 갱신하고 재사용해야 함. (`recover-session:92-103`)

- [x] **[C4] queue pop 비원자적** — `queue_pop()` 후 tmux 생성 실패 시 작업 소멸.
  동시 `--from-queue` 두 호출이 같은 head 선택 가능. (`spawn-worker:109`, `lib.sh:157`)

- [x] **[C5] state.json 죽은 상태** — `lib.sh:56` 초기화만 있고 실제 worker 추가 코드 없음.
  `spawn-worker:204`의 "Update state.json" 주석만 존재. (`spawn-worker:204`)

- [x] **[C6] --auto-fix tmux 없을 때 파괴적** — `tmux list-windows` 실패를 빈 배열로 처리해
  임시 tmux 중단 시 모든 worker를 `stale`로 마킹. 세션 존재 여부 먼저 확인 필요.
  (`recover-session:48`)

## High (기능 완전히 안 됨)

- [x] **[H1] --split-log 두 가지 버그**
  (a) `log_file`에 worker stdout 실제로 안 들어감 — spawn event만 기록. (`spawn-worker:202`)
  (b) `split-window` 후 bottom pane이 active → worker 명령이 `tail -f` pane으로 전달됨. (`spawn-worker:138, 198`)

- [x] **[H2] capture-worker stdout 오염** — `log_success`/`log_info`가 stdout으로 나와
  pane 내용과 섞임. monitoring loop 파싱 불가. stderr로 이동 필요. (`capture-worker:65,69`)

- [x] **[H3] capture-worker pane 비결정적** — window 대상 capture; `--split-log` 상태면
  어떤 pane이 active냐에 따라 log pane을 캡처할 수 있음. pane index 명시 필요. (`capture-worker:68`)

- [x] **[H4] queue.json 동시성 없음** — `queue_push()` lock 없이 read-modify-write.
  두 호출이 같은 `q-###` 생성 → 마지막 `mv`만 살아남아 하나 소멸. (`lib.sh:122`)

## Medium (운영 안정성)

- [x] **[M1] SESSION_NAME 하드코딩** — `lib.sh:47`, `spawn-worker:132`에 `sora-backend` 리터럴.
  `REPO_ROOT basename` 기반으로 동적 생성 필요. 다른 clone 사용 시 세션 오염.

- [x] **[M2] validate_worker_json() 미사용** — `lib.sh`에 존재하나 어디서도 호출 안 함.
  손상된 worker 파일 하나가 `set -e`로 전체 tool 종료시킴. (`recover-session:57`, `dashboard:78`)

- [x] **[M3] dashboard watch exec quoting** — `watch -n5 "... --once"` 방식은 따옴표 중첩 문제.
  `watch -n5 sh -c '... --once'` 패턴으로 변경 필요.
