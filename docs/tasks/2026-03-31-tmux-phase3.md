### 2. SKILL.md 확장

#### 2.1 오케스트레이션 행렬에 6번째 경로 추가

기존 5개 경로 이후에 추가:

```md
### 6. Existing active task with worker dispatch state

Use this when `docs/current.md` shows `하네스 상태 == "worker-dispatch-ready"` and `HAS_TMUX_ORCHESTRATION=1`:

- `start-harness` is in "parallel worker delegation" mode
- Read `docs/current.md` for `활성 워커` list (count of workers needed)
- Call `list-workers --json` to get current state
- For each pending subtask:
  - Call `spawn-worker codex <slug> <task-ref> --spec docs/tasks/<subtask>.md --auto-start`
  - Record worker ID
- **Monitoring loop** (while workers not all done):
  - Call `list-workers --json` every 30 seconds
  - Count workers with status != "done"
  - If any worker is "failed" or "blocked":
    - Call `capture-worker` to get last 50 lines
    - Log diagnosis, pause monitoring
    - Request Claude decision
  - If all workers done:
    - Continue to PR creation/merge steps
    - Call `recover-session --auto-fix` to normalize state
    - Update `docs/current.md` (remove `활성 워커` field)
```

#### 2.2 기존 "Orchestration Matrix" 섹션 앞에 조건 추가

### 3. 병렬 워커 위임 흐름의 의사코드

SKILL.md에 포함할 흐름:
```
PARALLEL_WORKER_DELEGATION_FLOW:
  1. read docs/current.md for active spec and 활성 워커 field
  2. parse 활성 워커 to count expected workers (e.g., "worker-001, worker-002, worker-003" → 3)
  3. while true:
       3a. list-workers --json → parse current workers
       3b. pending_workers = workers with status "starting" or "running"
       3c. spawned_workers = count(workers)
       3d. if spawned_workers < expected_count:
             spawn missing workers via spawn-worker --auto-start
       3e. for each worker:
             if worker.status == "failed" or "blocked":
               capture-worker <worker-id>
               pause monitoring
               request Claude decision
             if worker.status == "done":
               continue
       3f. if all workers "done":
             break loop
       3g. sleep 30 seconds
  4. recover-session --auto-fix
  5. continue to PR creation/merge steps
```
